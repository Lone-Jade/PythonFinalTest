## 一、问题分析

### 1.1 泛化瓶颈

| 问题 | 证据 |
|------|------|
| 训练集最大实例 | 20×10×3 = 200 任务 |
| 测试集最大实例 | 100×10×6 = 1000 任务 |
| 规模差距 | **5×** |
| Dueling DQN 在 100×10 | 落后启发式 25-30% |

### 1.2 根本原因

原始特征包含规模相关的绝对值：
- `time / t_ref` — t_ref 随实例变化
- `job_id / n_jobs` — 比值在小/大实例分布不同
- `remaining_ops / n_machines` — 同

普通 MLP 学习到的是**与训练集规模耦合的特征表示**，无法泛化到未见过的规模。

## 二、ScaleInvariantDuelingNetwork

### 2.1 架构

```
Input: [global(6), worker(4), action(9)] = 19 dims
    │
    ▼
LayerNorm(input) ─── 跨特征维度归一化，消除规模差异
    │
    ├── StateEncoder (10 dims):  Linear→LN→ReLU→Linear→LN→ReLU → state_emb (128d)
    │
    ├── ActionEncoder (19 dims): Linear→LN→ReLU→Linear→LN→ReLU → action_emb (128d)
    │
    ├── ValueHead:     state_emb → Linear → V(s)
    │
    └── AdvantageHead: concat(state_emb, action_emb) → Linear→LN→ReLU→Linear → A(s,a)

Q(s,a) = V(s) + A(s,a) - mean(A)
```

### 2.2 关键设计

| 组件 | 作用 | 规模不变性贡献 |
|------|------|:--:|
| `LayerNorm(input)` | 归一化 19 维输入特征 | ⭐⭐⭐ 消除绝对值差异 |
| 分离 State/Action 编码器 | state 和 action 独立处理 | ⭐⭐ 防止规模信息泄漏 |
| Pre-LN 隐藏层 | 每层前 LayerNorm | ⭐⭐ 稳定深层训练 |
| State-Action Fusion | 拼接后融合 | ⭐ 学习 state-action 交互 |

### 2.3 参数对比

| 网络 | 参数量 | LayerNorm | 分离编码 | Dueling |
|------|:--:|:--:|:--:|:--:|
| PairScoringNet | 37,250 | ❌ | ❌ | ❌ |
| DuelingPairScoringNet | 74,500 | ❌ | ❌ | ✅ |
| **ScaleInvariantDueling** | **71,464** | ✅ | ✅ | ✅ |

> 比 Dueling 少 4% 参数但包含更多归一化层

## 九、分析

### 9.1 LayerNorm 为何有效

```
不同规模实例特征分布:
  小实例 (6x6):  time/t_ref ~ 0.01-2.0,   job_id/n_jobs ~ 0-1.0 (粗粒度)
  大实例 (100x10): time/t_ref ~ 0.1-50.0,  job_id/n_jobs ~ 0-1.0 (细粒度)
  
LayerNorm → 无论输入来自哪种规模，都映射到 μ=0, σ=1 的分布
→ 网络学到的是相对关系，不是绝对值
```

### 9.2 分离编码器的价值

- **State Encoder**: 只处理全局+工人特征，学习"当前状态好坏"
- **Action Encoder**: 处理完整特征，学习"这个动作在当前状态下好不好"
- **Fusion**: 组合两者，学习"相对于其他动作，这个动作应得多少分"

这种显式的 state→action 分解，让网络更容易学到"比较"而非"记忆"。

### 9.3 尚存问题

1. **2-worker 实例差**: 100x10x2 仍落后 24%，2-worker 场景工人切换频繁
2. **训练集仍缺大实例**: 最大训练仅 20x10×3，要达到启发式水平需入训大实例
3. **早期 plateau**: ep=100 即达最佳，后续无改善
