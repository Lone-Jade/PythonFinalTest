**修改**: Prioritized Experience Replay（优先经验回放）

---

## 一、修改内容

### 1.1 新增 TrainConfig 参数

在 `config.py` 的 `TrainConfig` 中新增 3 个 PER 参数：

| 参数 | 值 | 含义 |
|------|:--:|------|
| `per_alpha` | 0.6 | 优先级指数（0=均匀采样，1=完全按优先级） |
| `per_beta` | 0.4 | 重要性采样权重退火起始值 |
| `per_eps` | 1e-6 | 避免零优先级的小常数 |

### 1.2 PrioritizedReplayBuffer 实现

替换原有的 `ReplayBuffer`，新增 `PrioritizedReplayBuffer`（`agents.py`）：

- **存储**: 每个 transition 附带一个 priority 值
- **采样**: 按 `P(i) ∝ priority_i^α` 的概率分布采样
- **重要性采样权重**: `w_i = (N × P(i))^(-β)` 归一化，修正非均匀采样偏差
- **β 退火**: 随采样次数从 0.4 线性增长到 1.0（50000 帧）
- **优先级更新**: 每次 `update()` 后用 `|TD-error| + ε` 更新被采样 transition 的优先级
- **新经验**: 以当前最大优先级入队，确保新经验被充分学习

### 1.3 DQNAgent.update() 修改

```python
# 旧: batch = buffer.sample(batch_size)
# 新: batch, indices, is_weights = buffer.sample(batch_size)
#     loss = (weights * smooth_l1_loss(pred, target, reduction='none')).mean()
#     buffer.update_priorities(indices, td_errors)
```

### 1.4 环境保持不变

Reward Shaping 参数（`s_job_completion=0.5`, `s_efficiency=0.05`, `s_progress=0.02`, `s_stall=0.1`）保持不变。PER 仅修改 DQN 的经验采样方式。

## 七、分析

### 7.1 PER 为何有效

1. **打破均匀采样局限**: 原始 ReplayBuffer 均匀采样，大量"平庸"经验（已学会的决策）稀释了高价值经验
2. **高 TD-error 优先**: 代理预测误差大的 transition 优先回放 → 更快修正错误认知
3. **重要性采样修正**: IS 权重确保梯度估计无偏，避免优先级引入的分布偏移
4. **大实例 + 多经验**: 大规模问题经验池更丰富，优先级筛选的增益更大

### 7.2 PER 与 Reward Shaping 互补

| 维度 | Reward Shaping | PER |
|------|:--:|:--:|
| 作用层 | 环境（奖励信号） | 算法（经验采样） |
| 受益方 | PPO > DQN | DQN >> PPO |
| 原理 | 密集化奖励信号 | 优先化经验学习 |
| 组合效果 | 两者各自发挥优势，DQN 重新主导 | |

### 7.3 尚存问题

1. **小实例退化**: PER 在小实例上偶尔比无 PER 差（6x6x2, 10x10x6）
2. **PPO 稳定性**: 仍有验证崩溃（ep=20, makespan=668055）
3. **未稳定超越启发式**: 大实例上差距仍为 5-15%
4. **PER 超参**: α=0.6, β 退火速度可能非最优
